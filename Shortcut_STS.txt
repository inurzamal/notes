** Ctrl + Shift + L : To see shortcut keys in STS
--------------------------------------------------

1) Ctrl + Shift + R : To search for a particular file (use *filename start with)

2) Ctrl + Shift + T : To find classes in the application or predefined classes present in the jars(dependencies)

3) Ctrl + W : To close the opening file

4) Ctrl + Shift + W : To close all the opening file

5) Ctrl + M : Maximize file or restore

6) Ctrl + D : Delete the line

7) Ctrl + Shift + O : To import predefined class or To remove all un-used imports (organize the imports)

8) Ctrl + Shift + F : Format code

9) Ctrl + / : To comment or uncomment

10) Ctrl + Shift + / : Multiline Comment

11) Ctrl + Shift + \ : uncomment multiline

12) Ctrl + O : To display outline of the class (to see all the variables and methods) 
               - press again to see super class methods also

13) Ctrl + Shift + P : To see closing braket of the opening bracket

14) Ctrl + 1 : assign statement to a local variable

15) Alt + Shift + Z : surround with try catch (Note: Select the statements for try clause in the method )

16) Alt + SHift + R : Refactor/rename a variable or method (select the vaiable first)

17) Ctrl + space :  Autocompletion recommendations

18) Ctrl + F11 : Run

__________________________________________________________________________

    DEBUGGING
    ----------

    F11 (Fn+F11) : Debug

    F6 (Fn+F6): Step over, i.e execute and go to next line

    F5 (Fn+F5) : Step into- Go inside method body from method call.

    F7 (Fn+F7) : Step return- Come back to method call from method body

    F8 (Fn+F8) : Finish Execution, also used to go to next nearest break point

    Ctrl + Shift + I : Inspect (-> Select the expression/condition before inspect)

    Ctrl + Shift + B : Create/Remove breakpoint


  ** Note: If project has no breakpoints then debug ( Fn + F11 ) also behaves like run ( Ctrl + F11 )
